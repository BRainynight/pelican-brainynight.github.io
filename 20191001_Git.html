Ch1 入門與設定

三種狀態

  • committed 已遞交 : 進入本機資料庫
  • modified 已修改 : 改動檔案，尚未遞交資料庫
  • staged 已暫存 : 對修改的檔案目前的版本標識，將其加入下次要遞交的快照當中。

環境設定 : git config

每一級會覆蓋上一級的設定，因此.git/config優於/etc/gitconfig

  • /etc/gitconfig: 系統中所有的使用者與倉庫的值，git config --system從裡面調用，改寫設定變數
  • ~/.gitconfig or ~/.config/git/config : 針對自己，傳入 --global改寫
  • 當前目錄下的 .git/config: 針對單一倉庫。

使用者身分

每一次遞交都需要用到這些資訊，會被寫入遞交中，不可更改。

傳入--global ，就只需設定一次，如果在某特殊專案想改用其他用戶資訊，在該專案設定中，指令不要輸入 --global即可。

git config --global user.name "NAME"
git config --global user.email xxxx@example.com

個人編輯器 : use notepad

git config --global core.editor "'YOUR NOTEPAD.exe PATH' --multiInst "

檢視設定

git config --list #全部的值
git config <key> #單一key的值 ex. git config user.name

Ch2 基礎

把當前的目錄加入Git

git init #建立，但是尚未追蹤任何檔案
git add .
git commit -m"Your commit"

跳過暫存直接遞交

加上-a，如此一來不必再使用git add。

git commit -a -m"YOUR commit"

複製網路上的倉庫

git clone會將伺服器上的所有，包含歷史版本複製。可以在url後面，加上自己想更改的新專案名字。

git clone [URL] [MyProjectName (Optional)]

檔案操作

檢視倉庫中的檔案狀態.

+-- Tracked 已追蹤 | +-- 未修改 | +-- 已修改 | +-- 已暫存 +-- Untracked 未追蹤

  • Untracked 未追蹤 : 指上次遞交快照時，快照中沒有的檔案。Git 不會主動將他們放到下次的遞交名單，除非告訴Git需要追蹤。

  • git status : 檢視目前的檔案狀態

  • git status -s : 檢視目前的檔案狀態 (簡潔版)
    簡潔版的輸出格式為: (暫存與否)(修改與否) 路徑&檔案名
    A (已暫存)、M(已修改)、??(沒被追蹤的新檔案)、MM (已修改並已放到暫存區)

 M README #
?? LICENSE.txt
  • git diff : 哪些變更還沒有暫存? 顯示出尚未進入暫存的變更，輸出是更新(patch)) 如果所有的變更都進入暫存區，此指令不會輸出任何內容。
  • git diff --staged : 哪些已暫存的變更正在等待提交 。會將正在暫存的內容和上次遞交的內容做比較。
  • git diff --cached: staged 和 cached 是同義字，兩個指令等價。

忽略檔案

建立一個.gitignore的檔案，描述要掠過的檔案名稱。 參見 A collection of useful .gitignore templates

刪除檔案

從Git中移除某個檔案: git rm

  • 自行把檔案從工作目錄刪除，記得使用git rm告訴 git 別追了，否則該檔案狀態會處於 Changes not staged for commit。
  • 更改某個檔案，並且已經暫存: 使用-f強制移除。如此一來的資料無法透過 git 恢復。
  • 檔案保留工作目錄，但從暫存區移除 (不讓Git管理 但忘記寫在 .gitignore): git rm --cached filename
範例
git rm log/\*.log # 刪除log資料夾下，所有副檔名是 .log 的
git rm \*~ #刪除所有 「~」 結尾的檔案。

移動檔案

git mv file_from file_to

此指令相當於

mv file_from file_to
git rm file_from 
git add file_to

(其實直接給檔案改名，Git還是會知道)
