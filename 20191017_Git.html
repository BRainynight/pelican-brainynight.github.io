遠端

  • git remote顯示所有遠端倉庫的簡短名稱。
    origin: Git 給複製原始伺服器取的預設名稱。
  • git remote -v 顯示倉庫對應的URL

增加

git remote add [ShortName] [URL]

如此一來，可以在指令中，用 ShortName 代替完整的URL。

擷取

git fetch add [Remote-Name(上面的ShortName)]

取得所有本機倉庫中沒有的資料，只會單方向的把資料擷取到本機倉庫。 不會合併到本基的工作成果，也不會修改任何資料。 (需要合併的情況參見git pull)

複製倉庫時，指令會自動將遠端倉庫的位置取名 origin。

把資料推到遠端

預設: git push origin master
使用取的別名: git push [Remote-name] [Branch-name]

檢查

預設: git remote show origin
使用取的別名: git remote show [Remote-name]

會列出遠端倉庫的URL、每個分支的追蹤資訊等等。

改名

git remote rename [origin name] [new name]

刪除

git remote rm [remote-name]

標記

把特定歷史版本，標記成重要版本。

git tag # 列舉 

git tag -l"v1.8.5*" # 比對

建立註釋

git tag -a v1.4 -m"My commit" #如果-m後面沒有資訊，會開啟文字編輯器給使用者輸入。"
git show v1.4 # 可以看到註釋

建立標記

git tag [version_name]，不會包含其他任何資訊，為當前的版本建立標記。

為歷史紀錄補標記

  1. git log --pretty=online:
    可以看到所也得歷史紀錄，形式是 [校正碼(很長)] [內容]。

  2. git tag -a [version_name][部分校正碼]

    參見: Git 基礎 - 標籤

別名

設計別名，取代冗長的指令!

git config --global alias.ci commit # git ci 可以代表 git commit 
git config --global alias.lase 'log -l HEAD' #git last 代表 git log -l HEAD
git config --global alias.visual '!gitk' # 使用 git 為外部指令寫git 的所寫: git visual 會執行 git k

使用到外部指令時需要加上驚嘆號 「!」 (但我覺得不如寫 makefile ...)

分支

簡單來說就是 linkedlist 的感覺。 雜湊值(SHA-1)像是記憶體位置，分支(branch)像是慣用的 head、tail那些指向物件的pointer。 創建分支就是創建新的 pointer

使用者發起遞交時，Git 儲存 commit object (遞交物件): 指向暫存區快照的指標 (Pointer)、作者資料、遞交資訊、指向父遞交的指標。

  • 初始遞交沒有父指標
  • 對兩個或以上的分支合併遞交而言，存在多個父遞交(parent)。

名詞

  • bolb 物件 : Git中，檔案的目前版本資料。
  • tree 物件 : 紀錄目錄結構、bolb物件和檔案名稱的對應關係
  • 遞交物件 : 本次遞交的作者資訊、tree object、父遞交。

遞交的內部運作

假設: 當前資料夾為 ch3 ，其下有三個檔案 * ch3 * file1.txt * file2.txt * file3.txt

預存 (Stage)

  1. 對檔案的｢內容｣計算雜湊值(SHA-1)

  2. 把這個版本內容保存到 Git 版本庫 = blob object

  3. 將雜湊值寫入預存區

遞交(Commit)

使用git commit 時， 1. 計算每個子目錄的雜湊值 2. 將這些目錄記錄成 Tree object (紀錄資料夾結構、與雜湊值和節點的對應) 3. 遞交出去Commit object 時，除了作者之類的一些遞交資訊，還有指向 Tree object 的指標 (而Tree object 記錄了指向子物件的指標)。 4. 後續的遞交中，Commit object 裡會有一個 parent資料:指向上次的遞交物件。

Git 分支 (branch)

一個指向某次遞交的可移動輕量級指標，預設分支名稱是 master ，它會自動指向最後一次的遞交。

建立新分支

建立新的分支，但是仍在master分支上跑 。處在哪個分支由 特殊的pointer: HEAD決定。

HEAD指向 master表示在 master branch 上跑，指向 testing 表示在 testing branch 上跑。

git branch testing # 建立新分支
git log --oneline --decorate # 查看各分支指向的物件

切換分支

git checkout testing

當前分支，如果被切換到的是較舊版本，除了HEAD指標的切換，還會把工作目錄中的檔案恢復到該舊紀錄的快照狀態。

如果 Git 目前的狀態無法乾淨的完成恢復，不會允許使用者切換分支。

分岔

在兩邊的 branch 上各自做出更動，使得檔案紀錄出現分岔。使用git log --oneline --decorate --graph -all 會簡單的劃出分岔圖。

基本的分支操作

建立並切換到特定分支

git checkout -b iss53

等價於

git branch iss53
git checkout iss53

切換前要注意，如果工作目錄或暫存區存著尚未遞交(commit)的變更，這些未被記錄的東西和即將要切換到的分支衝突，Git 不會允許切換。

合併 (下載到本地)

fast-forward

在master branch上，合併 hotfix branch 的物件。

由圖可見，master分支指向的位置，是hotfix的上游。這個合併是單線的，沒有平行衝突 (沒有分歧)，稱為fast-forward。合併後master、hotfix 兩個 branch 都會指向 C4。

git checkout master 
git merge hotfix
git branch -d hotfix # 合併後不需要 hotfix branch，刪除。

基本合併(簡單的三方合併)

由圖可見，本次的合併從某個節點起有分岔，Common Ancestor = C2。

git checkout master
git merge iss53

Git會取用C2、C4、C5的快照為基礎，建立新的快照(C6)。合併完成後，創建一個新的commit object，它 (C6)被稱為合併遞交( merge commit)。

合併遞交的特殊性在於不只一個父遞交 (parent)。

衝突

如果兩條分支上，各自對同個檔案進行了變更，使得合併產生衝突。

Git 會暫停合併過程，等待衝突解決。

檢查哪些檔案沒有被合併(unmerged): git status

發送 (Push)

git push (remote) (bransh)

發送 本地分支「 serverfix 」到遠端伺服器上: git push origin serverfix

遠端伺服器會新增一個名為 serverfix 的分支

如果希望遠端的分支使用其他名字，例如 awesome

git push origin serverfix:awesome

Hint
如果使用https進行發送，git伺服器會每次都要求使用者名稱和密碼。(伺服器會判斷此使用者是否有許可權發送資料。 如果不想每次都輸入密碼，可使用git config --global credential.helper cache將憑證暫存在記憶體中 。

如果取得本機尚未追蹤到的遠端分支，Git 不會在本機提供該分支可編輯備份。也就是說，假設serverfix是遠端的新分支，git fetch origin時，雖然知道 origin/serverfix指向誰，但不會在本地創建名為serverfix的 branch (看得到吃不到的概念)。

  • 把該分支合併到目前的工作分支: git merge origin/serverfix
  • 建立本地的serverfix分支: git checkout -b serverfix origin/serverix
  • 希望本地分支不同名git checkout -b sf origin/serverix

追蹤

已遠端branch 為基礎建立的本地分支，預設會成為追蹤分支(tracking branch)，或稱作上游分支(upstream branch)。 追蹤狀態下，在當前分支git push即會直接推送到對應的追蹤分支上。
同樣的，git pull也會把遠端的資料，載到對應的本地分支。

git clone時，預設追蹤origin master->本地master。

建立並追蹤某分支

git checkout --track origin/serverfix

等價於git checkout -b serverfix origin/serverix，是此指令的簡化版。

事實上，如果執行切換操作(git checkout)到遠端已存在的分支(ex.serverfix)，而該分支尚未在本地建立。

git 會自動在本地建立新的同名分支，並追蹤該遠端分支。

已建立的分支上追蹤某遠端分支

使用-u或是--set-upstream-to選項 : 

git branch -u origin/serverfix

Hint
已建立好上游分支的情況下，上游分支可以用 @{u} 或是 @{upstream} 代替。 例如 git merge @{u} = git merge origin/master

檢視

檢視分支和追蹤清單 (此指令不會和遠端通訊)

git branch -vv

如果想知道，本地分支和伺服器分支的進度相差多少。

git fetch --all;git branch -vv

擷取

git fetch只會擷取本機沒有，並更新這些資料。但是不會更改工作目錄，依然需要手動合併

git pull原則上、大多數情況下等同於git fetch之後執行git merge。

刪除

git push origin --delete serverfix
