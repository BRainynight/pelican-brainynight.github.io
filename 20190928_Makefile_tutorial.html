Makefile的說明

夜雨 | 2019-09-28, 周六

前言

Make 是什麼?

Make是一支程式，就好像pip, python一樣。
如果直接在CMD打python，會進入python的執行畫面。

直接在CMD輸入make，會呼叫出那個叫做make的應用程式。這後續再提。

make在linux中是一個非常好用的東西。 他的原理是在當前目錄下，搜尋名為 Makefile (或是 makefile) 的文字檔。 這個文字檔中，會記錄一些如何執行動作的「懶人包」。

在Windows上安裝Make

由於make是專屬Linux的東西，在windows上需要做一些「近似」。 像是安裝gcc編譯器...等等等的。
這裡記錄一下我為了Make執行跟安裝的動作，因為我的電腦之前搞不好有裝過其他東西，有些環境不必再安裝。
1. 下載MinGW
2. 安裝MinGW (C:\MinGW)
3. 增加環境變量
控制台→ 系統及安全性 →系統 → 進階系統設定 →環境變數→系統變數。
在變數欄中找到「Path」→點選Path之後，按編輯 → 新增 → C:\MinGW\bin
4. 修改make名字:為了跟linux保持一致。
複製C:\MinGW\bin\mingw32-make.exe，改名稱為make.exe
5. 在CMD輸入 make -v，若有顯示當前版本信息，表示安裝成功
資料來源:windows像linux般使用gcc,make等工具

ERROR

致命錯誤: unknown command

fatal error: unknown command

這個問題困擾我許久，尋找解法也未能改善。因為我的C槽容量不夠，幾乎所有的安裝都裝在D。
但這個問題始終沒有解決，最後索性重新安裝MinGW，按照他的建議裝在C槽。重新跑 mingw-get-setup 的應用程式，並且把所有的選項勾起來，安裝!

basic setup → 把全部的選項勾起(右鍵→Mark for Installaion )→左上角(Installation)→Apply Changes.

等他安裝完畢，總算解決這個ERROR!!

缺少Tab

Makefile:41: *** missing separator.  Stop.

查看 Makefile文件的第41行。這個問題是:指令前必須縮排，而縮排必須是tab。

我使用brackets作為編輯文件，右下角的設定切換成了「tab長度:4」(表示使用tab作為縮排。)
把41行前的縮排刪掉，重新使用tab縮排，就成功了!

寫Makefile!!

cd

Makefile 的 action section 中每一行 command 是會呼叫不同 shell instance,
因此, 如果要讓 Makefile 中執行 cd 切換資料夾再執行後續的指令，則要把 commands concat 在一起, 用 ';' 隔開 在makefile做cd動作
但我依然出現

cd D:/pelican/go/to/my/goal/dir ; git add .; git commit -m 'your comments' ; git push
系統找不到指定的路徑。

最後發現，合併commands還有另一個寫法:不使用";"，而是"&&"。
在Makefile中寫(&&前後是空白鍵)

github: publish    
    cd $(OUTPUTDIR) && git add . && git commit -m 'your comments' && git push

error: pathspec XXX did not match any file(s) known to git

cd D:/pelican/go/to/my/goal/dir && git add . && git commit -m 'your comments' && git push
error: pathspec 'comments'' did not match any file(s) known to git
Makefile:77: recipe for target 'github' failed

改使用雙引號!! 

github: publish    
    cd $(OUTPUTDIR) && git add . && git commit -m "your comments" && git push

注意，cmd在執行git的過程中，不要變更檔案!
(像是發現忘了儲存，趕快補存之類的...)，乖乖中斷終端機，儲存檔案後重新執行一次指令吧!!
